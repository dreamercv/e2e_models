# 检测训练排查与微型过拟合 — 分步调试手册

本文档汇总「loss 不降 / 看不到检测结果 / 数据很少」时的排查顺序，以及 **微型过拟合** 的具体做法，便于你按步骤执行。

---

## 第一部分：先建立正确预期

### 1.1 架构是否可行

「2D 特征 → BEV 几何聚合 → 时序融合 → BEV 上稀疏 query/instance」与 BEVDet、Sparse4D 等思路同类，**不能**仅凭「1 个 clip、很多 epoch 仍无框」就断定架构错误。更常见原因包括：

- 数据量与场景多样性不足  
- 推理阈值过严、或后处理滤掉所有框  
- 匈牙利匹配 / 监督 mask / 坐标系与实现不一致  
- 学习率、AMP、梯度等训练稳定性问题  

### 1.2 单 clip 训练的局限

- **分布极窄**：同一路段、相似光照与车流，query-based 检测仍需要足够的实例与几何变化。  
- **等效多样性不足**：300 帧 × 滑窗 ≠ 多场景大数据。  
- **适用场景**：单 clip 适合 **debug 管线是否通**，不适合下「架构是否成立」的结论。

### 1.3 从 loss 曲线读信号（结合你方 `loss.log` 形态）

| 现象 | 可能含义 |
|------|----------|
| `det3d_loss_cls` 很快从 ~1 降到 ~0.1～0.17 后**长期平台 + 抖动** | 多数 query 仍偏背景，或正样本/匹配仍弱 |
| `det3d_loss_box` 从很大降到中等（如 7～20）后**不再明显下降** | 回归在学但未贴准；或尺度/坐标系有系统偏差 |
| 整体 **非持续下降** | 数据上限、局部极小、或超参/实现问题 |

loss 有下降 ≠ 推理一定能看到框；需结合 **阈值、匹配数、坐标** 一起看。

## 第二部分：推荐排查顺序（按步执行）

完成一步再进入下一步；在某步发现异常，先修该步及上游。

### 步骤 1：区分「没有框」与「框被滤掉」

**目的**：避免把阈值/后处理问题误判为「模型完全没学」。

**做法**：

1. 推理时打印 **分类 score 的最大值、或 histogram**。  
2. **临时大幅降低 score 阈值**（甚至接近 0），或暂时关闭部分 NMS/过滤。  
3. 若阈值一降就出现大量低质量框 → 优先调 **校准、阈值、sigmoid/温度**；若仍完全无框 → 继续步骤 2。

---

### 步骤 2：检查匹配与正样本

**目的**：确认监督信号真的打在 query 上。

**做法**：

1. 训练 `forward` 后打 log：**匈牙利（或等价）匹配到的正样本数量**。  
2. 检查是否存在 **整 batch 匹配数为 0** 或 **GT 恒为空** 的 batch。  
3. 若长期几乎无正样本 → 查 **assign 代价、mask、类别数、空 GT 处理**。

---

### 步骤 3：坐标系与 BEV 范围

**目的**：排除「框在别处」或 loss 永远下不去。

**做法**：

1. 确认 GT 与 `grid_conf`、ego、相机外参、head 输出定义 **同一套坐标约定**。  
2. 可视化：**GT 投到 BEV 或图像**是否与标注一致。  
3. 单 clip 若有一处系统偏差，box loss 易长期偏高。

---

### 步骤 4：数据规模（正式训练）

**目的**：验证「泛化检测」需要的数据量。

**做法**：

- 微型过拟合通过后，再 **增加 clip 数与场景多样性**（远多于 1 段 300 帧）做正式训练。  
- 单 clip 只适合 debug，不作为最终结论依据。

---

### 步骤 5：优化与数值稳定

**做法**：

- box loss 长期高位震荡：尝试 **更小学习率、warmup、梯度裁剪**。  
- 对比 **关闭 AMP** 是否更稳（便于排除溢出/NaN）。  
- 权重是否已 NaN：见 `docs/BEV_ALIGN_NAN_DEBUG.md` 中参数检查思路。

---

### 步骤 6：BEV / 时序分支专项

若怀疑 backbone 或对齐：

- 参考 `docs/BEV_ALIGN_NAN_DEBUG.md` 对 `x`、`theta_mats`、`cam_pos_embed`、`grid` 做 **isfinite** 与分段断言。  
- 临时关闭 `use_cam_pos_embed` 做对比实验。

## 第三部分：微型过拟合实验（强烈推荐）

**目标**：用 **极少样本** 验证「模型 + loss + 匹配 + 坐标」能否 **记住** 数据。若微型都拟合不了，优先查实现与标签，而不是加数据。

### 3.1 需要多少数据

| 目的 | 建议数据量 |
|------|------------|
| 只验证检测头、loss、匹配是否自洽 | **1 帧**，batch 内 **1～2 个样本** 即可；GT **1～5 个框** |
| 带时序（如 seq_len=5） | **1 段连续窗口**（如 5 帧），或再加 **1～2 段不同窗口** 防偶然 |
| 稍稳、避免单次初始化运气 | **4～8 帧**，或 **2～4 张不同帧**（仍属「微型」） |

要点：**个位数帧 / 个位数样本** 即可；常见做法是 **1 个 batch 的数据重复训练几千～几万 step**。

### 3.2 实验设置清单

- [ ] **关闭** 随机数据增强（翻转、颜色、crop 等）。  
- [ ] 使用 **固定随机种子**（可选，便于复现）。  
- [ ] **学习率**：可略大于正常训练，或先用正常 lr 试；以 loss 能持续下降为准。  
- [ ] **BatchNorm**：若 batch 极小，可考虑 `eval()` 冻结 BN 或改用 GroupNorm/LN（仅用于本实验对比时）。  
- [ ] dataloader **只返回** 选中的 1 帧或 1 段窗口（或子集 `Subset`）。

### 3.3 怎样算「成功」

同时满足：

1. **训练 loss** 在微型集上能 **压到很低**（分类趋近 0、box 按你的定义也明显变小，具体量级与损失定义有关）。  
2. **同一批训练输入**上做 inference，**阈值调低**，能看到与 GT **大致重合**的框。

### 3.4 怎样算「失败」及下一步

- **≤8 帧、无增强、步数已足够（如 2k～10k+ step）**，loss 仍居高不下或推理仍无任何框：  
  - 优先查：**标注加载、mask、匹配、head 维度、loss 权重、坐标系**。  
  - 不要先归结为「数据还不够」。

### 3.5 与「1 个 clip 300 帧」的区别

| 微型过拟合 | 单 clip 长视频 |
|------------|----------------|
| **刻意极少帧**、可控、关增强 | 场景仍单一、但帧数多 |
| 验证 **能否记住** | 验证 **泛化** 不足 |
| 实现/标签 debug 的黄金标准 | 不能替代多场景数据 |

## 第四部分：建议的整体调试路线（一页纸）

1. **微型过拟合**（1 帧或 1 个 5 帧窗口）→ 不通则修实现/标签。  
2. **推理阈值与 score 分布** → 排除「有框被滤掉」。  
3. **训练 log：匹配正样本数** → 排除「无监督」。  
4. **坐标与可视化** → 排除「框在错误空间」。  
5. **扩数据 + 正式超参** → 再谈泛化。  
6. **NaN / Inf** → 见 `docs/BEV_ALIGN_NAN_DEBUG.md`。
